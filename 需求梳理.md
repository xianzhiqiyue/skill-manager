# AI Skill 管理工具链需求梳理

## 1. 核心目标

建立**"本地管理工具 + 云端注册中心"**体系，打破 Claude Code、GitHub Copilot、Codex、Cursor 和 Antigravity 之间的技能（Skills/Rules）隔离，实现**"一次编写，到处同步"**。

---

## 2. 技能标准定义 (Source of Truth)

所有技能统一采用 **SKILL.md** (Agent Skills) 开放标准：

| 组成部分 | 说明 |
|---------|------|
| 目录结构 | 每个技能为一个独立文件夹 |
| 元数据 | 必须包含 YAML Frontmatter（name, description） |
| 执行层 | 支持可选的 `scripts/` (Python/Bash) 和 `references/` 文件夹 |

---

## 3. 本地管理工具需求 (CLI Manager)

该工具负责物理文件的发现、分发与格式转换。

### 3.1 目录自动发现
扫描项目根目录至 Git 根目录的所有配置文件夹（如 `.claude/`、`.cursor/`、`.agent/`）。

### 3.2 统一映射机制 (Mapping)

**核心逻辑**：采用符号链接 (Symlink) 或物理同步。

| 工具 | 项目级映射路径 | 全局级映射路径 |
|-----|---------------|---------------|
| Claude Code | `.claude/skills/` | `~/.claude/skills/` |
| Copilot | `.github/skills/` | `~/.copilot/skills/` |
| OpenAI Codex | `.agents/skills/` | `~/.agents/skills/` |
| Antigravity | `.agent/skills/` | `~/.gemini/antigravity/skills/` |
| Cursor | `.cursor/rules/` | N/A（集成设置面板） |

### 3.3 智能转换器 (Transpiler)
- 当同步到 Cursor 时，自动将 SKILL.md 转换为 `.mdc` 格式
- 自动从 SKILL.md 提取描述信息填充到 .mdc 的 description 字段

### 3.4 冲突解决
遵循优先级：**项目级技能 > 全局级技能**

### 3.5 双模模式（Codex 建议）
由于 Cursor 和 Claude Code 对符号链接的支持可能因版本更新而失效：
- **优先尝试**：Symlink
- **自动降级**：若检测到 IDE 无法读取，则切换为"文件物理镜像同步（Mirror Sync）"

---

## 4. 云端注册中心需求 (Registry Server)

类似 npm，负责技能的版本化托管。

### 4.1 包管理规范

| 规范项 | 说明 |
|-------|------|
| 命名空间 | 支持 `@username/skill-name` |
| 版本控制 | 强制执行语义化版本 (SemVer) |

### 4.2 API 接口

| 接口 | 方法 | 功能描述 |
|-----|------|---------|
| `/search` | GET | 支持根据 description 进行语义搜索 |
| `/publish` | POST | 接收本地工具上传的技能压缩包（Tarball） |
| `/install/:name` | GET | 下载特定版本的技能包 |

### 4.3 安全审计 (Security Gate)

| 审计类型 | 说明 |
|---------|------|
| 静态扫描 | 自动检测脚本中的恶意命令（如 `rm -rf /`） |
| 意图检查 | 识别明显的提示词注入（Prompt Injection）攻击模式 |

---

## 5. 执行与协同流程 (User Flow)

```
┌─────────────────────────────────────────────────────────────┐
│  1. 开发技能                                                  │
│     用户在本地 ~/.agent-skills/ 开发一个新的代码审查技能         │
└──────────────────────┬──────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 推送云端                                                  │
│     执行 skill push，注册中心验证通过后存入数据库并发布版本       │
└──────────────────────┬──────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 多工具同步                                                │
│     在另一个项目中执行 skill sync --all                       │
└──────────────────────┬──────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 自动注入                                                  │
│     工具自动在项目下创建：                                     │
│     • .claude/skills/code-reviewer/                          │
│     • .cursor/rules/code-reviewer.mdc                        │
│     • .agent/skills/code-reviewer/                           │
└──────────────────────┬──────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────────────┐
│  5. Agent 调用                                                │
│     无论用户唤起哪个 IDE，Agent 都能通过本地路径读取到相同的       │
│     专家级指令                                                │
└─────────────────────────────────────────────────────────────┘
```

## 6. 关键设计亮点

| 亮点 | 说明 |
|-----|------|
| **统一标准** | SKILL.md 作为单一事实来源 (Source of Truth) |
| **透明同步** | 用户无感知完成多工具配置 |
| **安全闸门** | 发布前自动审计脚本安全性 |
| **容错机制** | Symlink/物理同步双模式适配不同 IDE |
| **语义搜索** | 基于 description 的智能技能发现 |
| **命名空间** | 支持用户/组织级别的技能隔离 |
